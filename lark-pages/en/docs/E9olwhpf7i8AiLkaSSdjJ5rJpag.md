# 2024-01-13 Release Notes

# 🎉 New Features

- Added: Pre-settlement preparation is divided into Business Operation Checks and System Data Checks
    - Business Operation Checks primarily review business procedures, provide reminders, and may be approved based on the actual circumstances
    - System Data Checks primarily perform data validation; they cannot be manually approved, <b>if errors occur please contact Customer Service for assistance</b>
    - Note: The complete end-of-day procedures must be completed by January 13

<img src="/assets/MskYbLXKgolG7RxFK1DjXsLfpaf.png" src-width="3496" src-height="1734" align="center"/>

- Added: Real-time transaction synchronization feature
    - After ledger adjustments or similar operations occur, settlement entries can now be generated in real time (previously generated after operation data synchronization)
    - The Backend Transaction Positions and Backend Transaction Funds pages have been decommissioned; position transaction and fund transaction views now display settlement data
    - Settlement entries are now based on real-time synchronization
    - In extreme cases, users can click the "Update Data" button to refresh settlement entries that have not been updated
