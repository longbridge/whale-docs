# Task Interface

# Assignment Interface

Whale provides basic user experience optimizations for unit assignments. The following are some common design interactions:

<b>Standardized batch-operation interaction on certain pages: corresponding action buttons appear after selection</b>

No records currently selected.

<img src="/assets/Ze1KbNn2wolYOlxQPwrjwrP4p3f.png" src-width="3328" src-height="1604" align="center"/>

After selection, applicable action buttons will automatically appear.

<img src="/assets/FATcbkvSeotrTnxPOvojbahIptg.png" src-width="3318" src-height="1614" align="center"/>

<b>Some unit-assignment table pages support clicking the full-row hot area to enter details (left-side slide-out)</b>

<img src="/assets/I0GZbR3C6oikZsxY047j92WFpRc.png" src-width="3352" src-height="1470" align="center"/>

Record details slide out from the left.

<img src="/assets/WpUJbS5IEo9REJxdcvPjKIZopig.png" src-width="3372" src-height="1620" align="center"/>

<b>Some unit-assignment pages support a bottom-anchored horizontal scrollbar</b>

Use the horizontal scrollbar at the bottom to view records to the left and right.

<img src="/assets/KCU1bFMUoo6OomxQ9MRjlw3upkg.png" src-width="3306" src-height="1519" align="center"/>

<b>Some unit table pages support a sticky table header (table columns remain fixed and do not scroll)</b>

<img src="/assets/Q9YVbBObPoe3vIxfQCNjLgXXp8b.png" src-width="3278" src-height="1245" align="center"/>

Table header is fixed at the top.
